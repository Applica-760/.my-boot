# 実行仕様書: CodeGraph Viewerのforce-directedレイアウト化

## 目的
- CodeGraph Viewerの描画をBFS階層レイアウトからforce-directed（力学シミュレーション）レイアウトに変更し、Obsidianのグラフビューに近い見た目・操作感を実現する

## 背景・前提
- 現在の描画ロジック（`scripts/codegraph-view-template.mjs` L89-160）はBFSで各ノードにlayerを割り当て、行ごとに等間隔に配置する階層レイアウトである
- 階層構造に依存するため、グラフの接続関係を直感的に把握しにくい
- Obsidianのグラフビューのような自由配置・インタラクティブな操作が望まれている
- 描画は生成されるHTMLファイル内で完結する（Node側での事前計算は行わない）

## 対象範囲
- `scripts/codegraph-view-template.mjs` — `generateHTML`関数内のHTMLテンプレート（レイアウト計算、ノード描画、エッジ描画、インタラクション）

## 非対象範囲
- `scripts/codegraph-view.mjs` — データ取得・graphData構築ロジックは変更しない
- graphDataのスキーマ（nodes, edges, rootIds等）は変更しない
- KIND_COLORS / EDGE_COLORS の色定義は変更しない
- CLIの引数・オプションは変更しない

## 変更要件

### レイアウト
- BFS階層レイアウト（layer計算→行配置）を全て削除する
- d3-forceによるforce-directedシミュレーションに置き換える
- 力のパラメータ: リンク引力、ノード間反発力、中心への引力、衝突回避を含める
- rootノードを初期配置の中心付近に置く

### ノード描画
- 現在のDOM要素（absolute positioned div）による描画方式を維持する
- シミュレーションの各tickでノードのDOM位置を更新する

### エッジ描画
- 現在のCanvas 2Dによる描画方式を維持する
- シミュレーションの各tickでCanvasを再描画する
- エッジの矢印表示を維持する

### インタラクション
- **ノードドラッグ**: 個別ノードをドラッグして位置を変更できる。ドラッグ中はそのノードの位置を固定し、シミュレーションを再加熱する。ドラッグ終了後は固定を解除する
- **パン**: 既存のビューポートドラッグによるパンを維持する。ノードドラッグとパンが競合しないようにイベントを分離する
- **ズーム**: 既存のホイールズームを維持する

### 外部依存
- d3-force関連モジュールをCDN（unpkg or cdnjs）から`<script>`タグで読み込む
- d3本体のバンドル（`d3`）を使うか、必要なサブモジュール（`d3-force`, `d3-dispatch`, `d3-timer`, `d3-quadtree`）のみ使うかは実装者判断とする

### ワールドサイズ
- 現在の事前計算による固定ワールドサイズを、ビューポートサイズ基準に変更する
- シミュレーションの中心をビューポート中央に合わせる
- Canvasサイズはビューポートに合わせ、pan/zoom変換に追従させる

## 実装方針
- `generateHTML`のHTMLテンプレート内のインラインJavaScriptのみを変更する
- BFS階層レイアウト部分（現在の`adj`, `layers`, `queue`, `layerGroups`, `sortedLayers`, `nodePos`計算）をd3-forceのシミュレーション初期化に置き換える
- tickハンドラ内でDOMノードの`style.left`/`style.top`更新とCanvas再描画を行う
- ノードドラッグは`mousedown`をノード要素で捕捉し、`d3.drag`相当の処理を自前で実装するか、d3-dragモジュールを追加で読み込むかは実装者判断とする
- 初期表示時にfit-to-viewportするロジックは、シミュレーション安定後（`end`イベント）に実行する

## 受け入れ基準
- [ ] 生成されたHTMLをブラウザで開くと、ノードがforce-directedレイアウトで配置される
- [ ] ノードを個別にドラッグで移動でき、周囲のノードがシミュレーションに従い再配置される
- [ ] エッジが矢印付きで正しく描画され、ノード移動に追従する
- [ ] パン（背景ドラッグ）とズーム（ホイール）が従来通り動作する
- [ ] rootノードが視覚的に区別できる（既存のroot強調スタイルを維持）
- [ ] レジェンド（Edge types）が従来通り表示される
- [ ] ノードのtooltip（hover時の情報表示）が従来通り動作する

## 検証条件
- 既存のCodeGraphプロジェクトに対して`node scripts/codegraph-view.mjs --project <path> --symbol <symbol>`を実行し、生成されたHTMLをブラウザで開いて以下を確認する:
  - ノードが階層ではなく力学的に分散配置されている
  - ノードをドラッグすると位置が変わり、接続ノードが引っ張られる
  - 背景ドラッグでパン、ホイールでズームが動作する
  - エッジの矢印がノード移動に追従する

## リスク・注意点
- CDNが利用不可の環境ではグラフが描画されない（d3の読み込み失敗時のフォールバックは非対象範囲とする）
- ノード数が多い場合（100+）、DOM要素の毎tick更新がパフォーマンスボトルネックになる可能性がある。現時点では最適化は非対象とする
- d3-forceのCDN URLはメジャーバージョンを固定し、破壊的変更を防ぐ
