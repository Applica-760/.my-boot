# タスク計画Skill

## 基本ルール
- ファイルパス規約: `docs/tasks/YYYYMMDD_HHmm_<topic>.md`
- 必ず下記の4セクション構成に従う
- 過剰な記述をしない。各項目は原則1行
- 原則 1md=1commit

## セクション構成
- **目的・前提・方針案**：（*ユーザが提供*）
- **計画**
- **実行ログ**
- **結果**

## 手順
1. 次に基づいて計画を立案
  - **目的・前提・方針案**
  - コードベースの実装
  - 指示に応じたWebSearch
2. 次をチャットに出力
  - 1.の計画概要
  - 情報不足、不明点
  - 意思決定が必要な点
3. ユーザの回答によって確定した計画を、[計画雛形](#計画雛形)に基づき**計画**に記録
4. Phase/Stepごとにチャット指示に基づき実装
5. 計画想定外の事象、バグに遭遇した場合、[ログ雛形](#ログ雛形)に基づき**実行ログ**に記録
6. 完了後、**結果**に記録し、`docs/tasks/done`にmdを移動する

## 雛形（原則各項1行）
### 計画雛形
- Phase/Step方式でToDoをチェックボックス`- []`管理。
- 原則1step 1行で、選定理由も記録
```md
## Phase 1:（Phase内容） 
- [x] （Step1内容）
- [] （Step2内容）
```
### ログ雛形
```
### 試行 N: <仮説>
- 実施内容:
- 結果:
- 判断: 続行 / 棄却 / 方針変更
```
