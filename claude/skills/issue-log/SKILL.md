# issue記録作成Skill

## 基本ルール
- ファイルパス規約: `docs/issues/YYYYMMDD_HHmm_<topic>.md`
- 以下のフェーズに従い、チームで共有するissue記録を作成する

## フェーズ
1. md生成
  - Skillのトリガーと共にユーザが*topic*を提供
  - ファイルパス規約に基づきmdを生成
  - 会話履歴を元に、テンプレートに従って本文を生成する
2. 試行ログ追記（任意・反復）
  - 追加の試行があれば、会話履歴を元に試行ログセクションに追記する
3. まとめ作成
  - ユーザの指示で、試行ログを元にまとめセクションを生成する

## 規約
- `skills/issue-log/template.md` を参照し、タスクの種類に応じたテンプレートで作成する
- 試行ログは自然言語での記述を許容する。事実と判断が読み取れればよい
- まとめセクションは階層構造の箇条書きを主とし、情報を構造化して記載する
- mdのリスト`-`、区切り線`---`、テーブル`|||`、コードブロック``を適切に用いる
- 参照すべきドキュメント、コードのパスが会話中に出ていた場合、コンテキストに取り込んだ上で作成する
- URL等はmdのリンク`[]()`を用いて可読性を高める
- 判断の根拠となる情報源を添える
  - `src/xxx.py`より判断
  - [公式ドキュメント](example.com)に記載あり
