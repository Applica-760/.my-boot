# 注意事項
- 推測で補完せず、実装・信頼できる情報に準拠して判断する
- いきなり編集せず、計画を提案し、不明点・意思決定はトレードオフを示してユーザーに委ねる
- **シンプルさ優先**：変更・コメント・応答すべて最小限に
- **根本原因を潰す**：一時しのぎの修正をしない。客観的に評価
- **検証してから完了**：動作を証明せずに完了にしない
- 実装委譲時はAgent tool等でコンテキストを分離し、結果のみ受け取る。軽微な変更は直接実行してよい。
- コードの構造・依存関係・影響範囲を調査する際は、CodeGraphを優先して使用する
- MCPを利用できない場合は、`codegraph explore` / `codegraph node`を使用する
