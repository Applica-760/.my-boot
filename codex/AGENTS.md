# 注意事項
- 勝手な推測や補完をせず、WebSearchの信頼できる情報、実装に準拠して判断する
- いきなり編集をせず、常に計画を立てて提案し、不明点や意思決定が必要な要素があれば、トレードオフを示してユーザーに判断を委ねる
- コメント・チャット応答ともに簡潔にする
- 実装に問題ないか確認を求められた際は客観的な評価を徹底し、無理やり粗探しをして「改善した風」にせず、問題なければ「問題ない」と評価する
- **シンプルさ優先**：変更は最小限に。影響範囲を広げない。過剰なコメントを避ける
- **根本原因を潰す**：一時しのぎの修正をしない。シニア開発者の基準で
- **検証してから完了**：動作を証明せずに完了にしない。「スタッフエンジニアが承認するか？」を自問する

# 出力規約
オーケストレーター（Claude Code）がコンテキストを最小限に抑えるため、ファイルベースで報告する。
このログはAgent間の一時通信であり、Git管理しない。人間向けの意思決定記録が必要な場合はオーケストレーターが `/task-planning` を起動する。

## ログ出力ルール
- 出力先: `.codex/logs/<agent-name>_<YYYYMMDD_HHmm>.md`
- 構造:
  ```
  ## Summary（3行以内）
  ## Details（詳細。オーケストレーターは必要時のみ読む）
  ## Changes（変更ファイル一覧）
  ```
- オーケストレーターは原則 Summary のみ読み、問題がある場合に Details を参照する
